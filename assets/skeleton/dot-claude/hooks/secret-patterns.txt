# SP1.5 secret-patterns.txt — shared by guard.mjs (PreToolUse redirection block) and
# secret-scan.mjs (PostToolUse detect-and-remediate).
#
# Format (one rule per line, whitespace-separated, exactly 3 fields):
#   <RULE_NAME>  <FLAGS>  <JS-regex-source>
# - RULE_NAME: no spaces; shown in output, never the matched secret bytes (F36).
# - FLAGS: "-" for none, or any combination of JS RegExp flags (commonly "i").
# - PATTERN: JS regex source (NOT POSIX ERE — this file is parsed by Node, not grep).
#   Patterns are applied per-line against file/payload text. Keep them bounded/anchored
#   to hold false positives down; use the allowlist (secret-allowlist.txt or an inline
#   `# pragma: allowlist secret`) for the rest.
#
# Lines starting with # and blank lines are ignored.
